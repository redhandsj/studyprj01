★ クリックやスクロールでDOMを生成するコンテンツはインデックスされない
 - Googlebotはjavascriptを実行することはできる
 - clickやスクロールのイベントは実行することができないため、イベントによって新しい要素が生成されるなかのコンテンツは読み取ることができません。
 - display:noneで隠すなど、初期表示の段階でHTMLに含まれていればインデックスの対象になる
   → !!!!! ただ、コンテンツの重要度は下がってしまうので気をつけましょう。 !!!!!

 - URLが変わらないと別コンテンツとして認識されないためGoogleにインデックスされなくなってしまいます。
   →　「１ページにコンテンツは１つだけ」／Googleが念押し　https://seopack.jp/seoblog/1url-1content/

★ 画像の利用を最適化する
 - alt属性の代替テキスト（altテキスト）を記述し、画像に関する情報を提供する

 - 画像は、一般的にサポートされているファイル形式（JPEG、GIF、PNG、BMP）を使う
 - 一般的なファイル名（image1.jpg、pic.gif、1.jpg）や極端に長いファイル名をつけない
 - 画像にリンクを貼る場合、alt属性を記述することで、リンク先ページの内容が理解されやすくな
 - 極端に長いaltテキストは避ける

 - 見出しタグをスタイル調整で使用せず、あくまで構造を示すために使用する

★ サイト外部からリンクを集める（被リンク）
 - 被リンク
   → あなたのブログのURLが、他のブログに記載してある状態
 - 発リンク
   → あなたのブログから、リンクが貼っている状態

 - あらゆるドメインからの被リンクを増やす
 - あらゆるアンカーテキストからの被リンクを増やす（アンカーテキスト分散）
   - アンカーテキストとは、HTMLソースでリンクが張られている文字列
 - !!! 被リンクを増やし、被リンクドメイン数も増やしながら、対策することが重要 !!!
 - あらゆるサーバーからの被リンクを増やす
   - あらゆるサーバー（IP）からの被リンクを増やすことで、被リンクの質を高めることが可能
 - 関連する被リンクを増やす
 - 発リンクの少ないページから被リンクを増やす

★ サイト内部からのリンクを集める（内部リンク）
 - 直接的なアンカーテキストからの内部リンクは避ける
 - リンク先がわかるようにすることが重要
 - 記事、コンテンツ文章に内部リンクを入れる
 - 内部リンクが構築できるコンテンツを増やす
   - 記事のテキスト量を増やす
   - 記事内のワードに関するコンテンツを作る
   - 人気記事コンテンツ等、自動生成コンテンツを増やす　etc…
 - 重要なページに向けて内部リンクを集中させる
   → 単語帳？

★ 低品質な被リンクを否認する
 - 【運用】順位を上げるために定期的に被リンクを見直し、低品質な被リンクを否認すること

オブジェクト指向の学び方・教え方

オブジェクト指向はなぜ難しい？

  • 基礎工学技術の勉強の必要性
  • 学習範囲が広い
  • 手続き型言語からの発想転換

オブジェクト指向への地図

  • 全プログラムはあるアルゴリズムとデータから成る
  • アルゴリズム→「こと」、データ→「もの」
  • 現実世界の「もの」に対応した高度な抽象的データ型の定義
  • データと処理を１つのプログラム単位とする（クラス）

アーキテクチャが鍵

  • データを中心にそれを処理するメソッドを作成
  • クラス間の関係を継承・集約により定義
  • データ中心の指向は、万能ではなく、機能の断片化を呼ぶ → 制御クラスなどの設置

オブジェクト指向分析を学ぶ

  • 上流工程からオブジェクト指向を適用すべき
  • UMLを用いて表す
  • 鍵は「抽象化」構造・機能・振舞の視点が大事。
